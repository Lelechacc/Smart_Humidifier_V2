Smart Humidifier V10

基于 ESP32-C3 与 FreeRTOS 的硬核全栈开发项目说明书
▶️ 实物全功能运行演示
展示内容：按键切换四种雾化模式、天问语音指令打断、WS2812B 状态同步灯效。

一、 具体实现了哪些功能？

🌫️ 多模态雾化控制
基于长/短按键，实现：待机(关闭PWM)、持续大雾、静音小雾、间歇喷雾(喷3秒停3秒) 四个状态机的精准流转。
🗣️ 离线语音绝对打断
接入 ASRPRO 离线语音模块。通过串口接收指令，可无视当前按键状态，直接通过语音口令跳转到指定模式。
🌈 状态同步流光灯效
通过 WS2812B 提供视觉反馈。例如大雾模式对应高亮色彩，间歇模式对应呼吸灯，实现功能与视觉的绑定。
📱 蓝牙参数预留接口
底层已初始化 BLE 蓝牙广播与协议栈，为后续开发微信小程序进行动态雾量曲线调节打下基础。

二、 硬件 BOM 与电路架构

🧠 主控与通信模块
MCU ESP32-C3FN4 (RISC-V架构)
内置 4MB Flash，原生支持 Wi-Fi 与 Bluetooth 5.0，负责全局 FreeRTOS 任务调度。
🔋 充放电与保护电路
充电 TC4056A 线性充电管理 IC。
保护 XB5306A 一体化锂电池保护芯片，提供过流、过放、短路三重物理保护。
⚡ 雾化高频驱动模块
驱动 AO3400 N沟道增强型 MOS 管。
配合大功率电感，承载 108kHz 的高频 PWM 信号，实现微孔雾化片的谐振。

三、 攻克了哪些底层技术痛点？

🛡️ MOS管烧毁与硬限位防御
痛点：108kHz 驱动下占空比过大会击穿 MOS 管。
解决：在底层 `bsp_atomizer.c` 中写入防御代码，强制将最高占空比物理锁定在 45% (寄存器值135)，无论上层怎么乱发指令，底层绝对安全。
🔄 GPIO矩阵解决走线冲突
痛点：语音模块强制 PA2 为 TX 端，与画好的 PCB 走线产生交叉死区。
解决：调用 ESP-IDF 的 `uart_set_pin` API，在代码层利用内部矩阵互换了 TX/RX 引脚的物理职责，免去飞线烦恼。
⏱️ 0 CPU 占用的 RMT 驱动
痛点：WS2812B 需要纳秒级时序，CPU 软件翻转会受 RTOS 调度打断导致闪屏。
解决：启用专用的 RMT 硬件外设接管数据发送，将色彩存入缓存区后由硬件自主生成波形，彻底解放 CPU。
